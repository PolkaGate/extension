# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please **do not open a public issue**. Follow the instructions below to report it safely.

### How to report

- Email security reports to: `support@ypolkagate.com`
- Use PGP encryption if possible. The public key is included at the end of this file.
- Include the following details:

#### Vulnerability details

- **Description** of the vulnerability
- **Steps to reproduce**
- **Impact assessment** (if known)
- **Affected versions**

### Response and Timeline

- We will **acknowledge your report within 48 hours**
- We will **provide updates** as we investigate and fix the vulnerability
- Fixed issues will be released in a **security update patch**
- You may receive **credit in the release notes** if you request

### What to Expect

- If your report is accepted, we will work with you to **coordinate disclosure responsibly**
- If your report is not a vulnerability, we will **respond with an explanation**
- **Do not attempt public disclosure** before a fix is released
